# 0043 - Domain Vocabulary And UI Language

## Status

Accepted

## Context

Users think in freelance-practice terms, not parser internals.

## Decision

Use domain words: inquiry, lead, client, budget range, follow-up, proposal scope, deliverable, payment terms, invoice line, tax category, payment match, contract party, clause, confidence, evidence, and anomaly.

## Consequences

Errors and explanations are easier to trust. Developer terms such as selector, schema node, parse token, and JSON path stay out of primary UI copy.

## Alternatives Considered

Generic data-import language was rejected because it makes the product feel like a spreadsheet utility instead of a consulting workflow.
